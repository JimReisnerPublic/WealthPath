# WealthPath

AI-powered retirement planning tool. Enter your financial profile and get an instant
retirement readiness score — powered by an XGBoost surrogate model trained on 50,000
households anchored to Federal Reserve SCF data.

## What It Does

- **Retirement readiness score** — probability of not running out of money, from an
  XGBoost model that approximates 500-path Monte Carlo simulation in ~1ms
- **SHAP-driven explanations** — shows the top 3 factors raising or lowering your
  probability (e.g. "your spending relative to Social Security is costing you 12 points")
- **Cohort benchmarking** — compares income and net worth against similarly-aged
  households from the Federal Reserve's Survey of Consumer Finances
- **AI planning agent** — LangGraph ReAct agent that calls all of the above as tools
  and writes a personalized retirement plan (requires Azure OpenAI)

## Stack

| Layer | Technology |
|---|---|
| Frontend | React + TypeScript + Vite + Tailwind + shadcn/ui |
| API | FastAPI + Pydantic v2 |
| ML model | XGBoost + SHAP (surrogate model) |
| Simulation | Monte Carlo (fallback + training data source) |
| Survey data | Federal Reserve SCF 2022 |
| AI agent | LangChain LCEL + LangGraph ReAct |
| Experiment tracking | MLflow |
| Live economic data | FRED MCP server |

## Quick Start

```bash
# Install Python dependencies
pip install -e ".[dev]"

# Configure environment (Azure OpenAI optional — app runs without it)
cp .env.example .env

# Generate training data and train the surrogate model
python scripts/generate_training_data.py --n 50000
python scripts/train_surrogate_model.py

# Terminal 1: start the API (from the wealthpath/ directory)
cd wealthpath
py -m uvicorn wealthpath.main:app --reload

# Terminal 2: start the frontend (from the wealthpath/ directory)
cd wealthpath/frontend
npm install
npm run dev
# → http://localhost:5173
```

Azure OpenAI credentials are **not required** to run locally. The `/plan/evaluate` endpoint
works fully without them. The `/chat/` endpoints degrade gracefully when unconfigured.

## API Endpoints

| Method | Path | Description |
|---|---|---|
| GET | `/healthz` | Liveness probe |
| GET | `/readyz` | Readiness probe |
| POST | `/api/v1/projections/` | Run Monte Carlo wealth projection |
| POST | `/api/v1/cohort/compare` | Compare against SCF cohort (survey-weighted) |
| POST | `/api/v1/plan/evaluate` | Retirement success probability + SHAP drivers |
| POST | `/api/v1/chat/explain` | LLM explanation of a projection result |
| POST | `/api/v1/chat/plan` | LangGraph agent — full retirement plan |

## ML Model

The surrogate model is trained offline on 50,000 synthetic households generated by
weighted-sampling from the SCF, then labeling each with a Monte Carlo success probability.
XGBoost learns the `household features → success probability` mapping and serves it at
inference in ~1ms vs ~200ms for Monte Carlo.

```
Training R² = 0.98    Test MAE = 3.4 percentage points
```

Top SHAP drivers: `net_replacement_rate`, `years_in_retirement`, `savings_as_income_multiple`

See [docs/learning/ml-model.md](docs/learning/ml-model.md) for details.
